# 领域文档 — 单上下文

本仓库使用**单上下文**布局：

```
/
├── CONTEXT.md          ← 领域术语表（概念与术语定义）
├── docs/
│   └── adr/            ← 架构决策记录（编号管理）
│       ├── 0001-示例.md
│       └── 0002-示例.md
└── src/
```

## 消费规则

以下技能在运行时需遵循：

1. **先读 `CONTEXT.md`** — 了解项目领域术语后再提出变更，避免发明新名词
2. **检查 `docs/adr/`** — 在触及的区域查找已有架构决策，不重复讨论已定事项
3. **术语一致性** — 全文使用 `CONTEXT.md` 中的词汇，不自行造词
4. **ADR 节制规则** — 仅在同时满足以下三条时才创建 ADR：
   - **难以逆转** — 事后改动的代价有实质影响
   - **缺乏上下文会令人困惑** — 未来读者看到代码会想「他们为什么这么做」
   - **真实权衡** — 确实存在有意义的替代方案，且你选了其中一个有特定理由
