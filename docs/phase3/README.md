# Phase 3 — 美术、音频与叙事资产管线

> 《七擒孟获》横版动作游戏
> 核心目标：建立可复用、可扩展、可替换灰盒的资产规范

---

## 1. 资产管线概述

Phase 3 与 Phase 2 并行推进。本阶段不是一次性做完所有美术，而是：

1. **制定统一规范** - 确保所有资产风格一致
2. **完成第1关必需资产** - 支撑第1关纵切从灰盒升级到正式美术
3. **建立7关扩展框架** - 为后续关卡开发提供模板

---

## 2. 像素三国美术规范

### 2.1 全局配色方案

| 用途 | 色值 | 说明 |
|---|---|---|
| 深藏青背景 | `#0F172A` | 夜空、主底色 |
| 近景黑蓝 | `#0B1120` | 前景剪影、暗角 |
| 深蓝灰 | `#1E293B` | HUD面板、山体 |
| 中蓝灰 | `#334155` | 远山、小地图 |
| 朱砂红 | `#DC2626` | 标题、边框、警示 |
| 亮红 | `#EF4444` | HP血条 |
| 琥珀金 | `#F59E0B` | 标签、标题辅助 |
| 金色高光 | `#FBBF24` | 光晕、火光、装饰 |
| 火橙 | `#EA580C` | 篝火、火星 |
| 翠绿 | `#22C55E` | 开始按钮、弹药条 |
| 奶油白 | `#FEF3C7` | 对话文字 |

### 2.2 角色尺寸规范

| 角色类型 | 基础尺寸 | 说明 |
|---|---|---|
| 小兵（玩家/蛮兵） | 32×32 或 48×48 | 常规角色 |
| Boss（孟获骑象） | 96×96 或更大 | 重要角色可更大 |
| 头像/立绘 | 128×128 或 145×120 | 对话框使用 |

### 2.3 Tile尺寸规范

| 类型 | 尺寸 | 用途 |
|---|---|---|
| 基础Tile | 16×16 或 32×32 | 地面、平台、装饰 |
| 大型装饰 | 64×64 或 128×128 | 背景、特殊地形 |

### 2.4 UI风格规范

- **边框样式**：像素边框 + 水墨纹理 + 朱砂红强调色
- **面板底色**：深木色 `#1C1917` 或 深蓝灰 `#1E293B`
- **文字颜色**：标题用琥珀金 `#F59E0B`，正文用奶油白 `#FEF3C7`
- **过场风格**：静态图文分镜优先，不做重动画

### 2.5 命名规范

```
格式：{角色}_{动作}_{方向}_{帧序号}.png
示例：player_idle_right_01.png / boss_menghuo_charge_left_03.png

英文snake_case优先，便于程序引用
```

## 5. 叙事资产数据结构

### 5.1 对话数据 (dialogs.json)
```json
{
  "id": "对话ID",
  "levelId": "所属关卡",
  "trigger": "触发条件",
  "speaker": "说话人",
  "portrait": "头像资源名",
  "text": "对话内容",
  "skippable": true,
  "firstPlayLockSec": 2.0
}
```

### 5.2 过场数据 (cutscenes.json)
```json
{
  "id": "过场ID",
  "levelId": "所属关卡",
  "trigger": "触发时机",
  "scenes": [
    {"type": "dialog", "data": "dialog_id"},
    {"type": "animation", "data": "anim_id"},
    {"type": "effect", "data": "effect_type"}
  ]
}
```

### 5.3 故事碎片数据 (fragments.json)
```json
{
  "id": "碎片ID",
  "unlockLevel": "解锁关卡",
  "title": "碎片标题",
  "content": "碎片内容",
  "category": "分类"
}
```

### 5.4 资产清单数据 (asset-manifest.json)
```json
{
  "id": "资产ID",
  "type": "sprite_sheet/tileset/ui/audio",
  "path": "资源路径",
  "frameWidth": 宽,
  "frameHeight": 高,
  "frames": 帧数,
  "usedIn": ["使用的关卡"]
}
```
