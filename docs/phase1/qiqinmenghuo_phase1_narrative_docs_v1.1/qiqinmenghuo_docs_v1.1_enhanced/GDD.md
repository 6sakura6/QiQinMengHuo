# 《七擒孟获》Game Design Document (GDD)

> 腾讯云黑客松 · 叙事类游戏赛道
> 版本：v1.1（叙事强化版） | 修订日期：2026-06-12
> 阶段：Phase 1 — 产品定义 & 叙事设计

---

## v1.1 修订目标

本版在 v1.0 的基础上重点补强三件事：

1. **定义更清晰**：把“魂斗罗式爽快射击”和“七擒七纵叙事循环”拆成可落地的设计原则，避免概念停留在口号。
2. **叙事更闭环**：让诸葛亮、孟获、玩家小卒三条成长线相互映照，确保每一关的战斗机制都能回答“为什么放走孟获”。
3. **开发更可执行**：补充数据结构、验收标准、风险规避、Phase 2 原型范围，方便后续直接进入 Phaser 原型制作。

## 1. 产品 Identity

### 1.1 一句话描述

**一款像素三国风格的横版动作游戏，以"擒而不杀、纵而再战"的七擒七纵循环为叙事引擎，让玩家扮演汉军无名士卒，在诸葛亮的计谋指引下，经历七场与蛮王孟获的宿命对决。**

### 1.2 核心体验主张

| 维度 | 定义 |
|------|------|
| **战斗感** | 魂斗罗式横版射击，爽快直接，像素子弹横飞 |
| **叙事感** | 七擒七纵的循环结构，每场 Boss 战结束才是故事的开始 |
| **文化感** | 三国智慧 + 蛮夷文化的立体呈现，非标签式古装 |
| **成长感** | 孟获从傲慢到折服的心态弧线，玩家亲身见证 |

### 1.3 差异化定位

> **大多数动作游戏的 Boss 打倒即结束。《七擒孟获》的独特性在于：打倒 Boss 只是故事的开始。**

- **"擒而不杀"机制**：每次战胜孟获后，诸葛亮下令释放，触发叙事节点
- **"纵而再战"递进**：下次相遇时孟获更强、更狡猾，关卡机制同步深化
- **小兵视角**：无名汉军士卒的视角带来更多沉浸和共情空间
- **AI 计谋台词**：接入腾讯混元大模型，根据玩家行为动态生成诸葛亮谋略分析

## 2. 核心机制设计

### 2.1 战斗系统

| 操作 | 描述 | 设计参考 |
|------|------|---------|
| 移动 | 左右横移，标准平台移动 | 魂斗罗 |
| 跳跃 | 单段跳 + 空中微调 | 魂斗罗 |
| 射击 | 八方向射击（上下左右 + 斜向） | 魂斗罗 / 合金弹头 |
| 闪避 | 短无敌帧翻滚（第2关解锁） | 死亡细胞 |

**武器系统（渐进解锁）：**
- **基础弩箭**（默认）：直线射击，中速
- **连弩**（第3关解锁）：三连发，射速快但单发伤害低
- **火箭**（第5关解锁）：点燃敌人，持续伤害
- **诸葛连弩**（第7关）：全方向散射，伤害最高

### 2.2 叙事系统

```
关卡流程 = 开场叙事(图文过场) → 关卡战斗(对话框穿插) → Boss战 → 擒获叙事(图文过场) → 释放抉择 → 过关解锁
```

**叙事介入方式（图文过场 + 对话框）：**

| 节点 | 形式 | 时长 | 可跳过 |
|------|------|------|--------|
| 每关开场 | 图文过场（1-2屏） | 15-30秒 | ✅ |
| 关卡中途 | 对话框（右上角弹出） | 不打断战斗 | ✅ |
| Boss战前 | 对话框（孟获台词） | 2-3秒 | ✅ |
| 擒获后 | 图文过场（2-4屏） | 30-60秒 | ⚠️首通不可跳 |
| 诸葛亮释放令 | 对话框 + 动画 | 10-15秒 | ✅ |

### 2.3 故事碎片收集系统

- 每关隐藏 1 块「历史碎片」
- 集齐 7 块 → 解锁完整史书记载 + 《出师表》相关段落
- 增加重玩价值，连接历史真实

### 2.4 AI 动态计谋系统（加分项）

```
玩家上关数据 → 行为分析（激进/稳健/快/慢/死亡次数）
              → 混元大模型生成诸葛亮计谋台词
              → 下关开场过场中展示
```

## 5. 画风定义：「像素三国」

### 5.1 核心风格

- **基底**：像素风（16×16 或 32×32 Sprite），保持魂斗罗式的爽快感
- **国风点缀**：
  - UI 边框：水墨笔触纹理
  - 标题/关卡名：毛笔手写字体
  - 过场插画：水墨渲染 + 像素角色
  - 粒子效果：墨滴飞溅替代传统火花
- **调色板**：低饱和 + 朱砂红 / 墨绿 / 古铜为主色调

### 5.2 各关环境色调

| 关卡 | 场景 | 主色调 | 氛围关键词 |
|------|------|--------|-----------|
| 第1擒 | 西洱河平原 | 翠绿 + 天蓝 | 开阔、初战、轻敌 |
| 第2擒 | 泸水河岸 | 深蓝 + 灰白 | 险峻、湍急、沉着 |
| 第3擒 | 紫荆山密林 | 墨绿 + 褐黄 | 压抑、伏击、智斗 |
| 第4擒 | 秃龙洞洞窟 | 暗紫 + 荧光绿 | 诡异、毒雾、坚持 |
| 第5擒 | 蛮族联盟营寨 | 赭红 + 土黄 | 宏大、联战、暗流 |
| 第6擒 | 蛮夷要塞 | 深红 + 铁灰 | 烈火、机关、反制 |
| 第7擒 | 银坑洞王庭 | 金 + 深棕 | 壮阔、决战、折服 |

## 6. 玩家体验目标

### 6.1 情感曲线

```
第1关：自信（这蛮王不过如此）
第2关：警觉（地形开始变难了）
第3关：压力（蛮兵变狡猾了）
第4关：挣扎（毒雾让人窒息）
第5关：被压迫（双Boss好难）
第6关：翻盘（我找到了克制方法）
第7关：敬畏（孟获全力以赴，值得尊敬）
结局：感慨（七擒七纵，心战为上）
```

### 6.2 首关体验承诺

- 玩家在 30 秒内学会移动+跳跃+射击
- 2 分钟内遇到第一个叙事节点（诸葛亮界面出现）
- 5 分钟内完成首次 Boss 战并触发"释放"叙事
- 完整流程 ≈ 8-10 分钟
- 目标：第一关结束后玩家必须感到好奇——「为什么要放他走？」

## 7. 叙事 × 玩法的融合设计

### 7.1 核心原则

> **每关的机制变化反映孟获的心态变化。**

```
孟获心态变化     →    关卡机制变化
────────────────────────────────────
第1擒：轻视汉军   →    基础教学关，孟获几乎不认真
第2擒：开始布局   →    引入环境障碍（水流）
第3擒：调兵遣将   →    敌兵种类增加，伏击机制
第4擒：联合外援   →    毒雾环境伤害，考验资源管理
第5擒：倾巢而出   →    多波次 + 双Boss
第6擒：出奇制胜   →    机关陷阱 + 巨型兵器
第7擒：背水一战   →    全机制综合，三阶段Boss
```

### 7.2 擒获后的叙事变体

每次擒获孟获后，叙事重点不同：

| 关卡 | 孟获台词核心 | 诸葛亮回应核心 | 孟获心态 |
|------|------------|--------------|---------|
| 第1擒 | "诡计！汉人狡诈！" | "将军勇则勇矣，然未识天时。" | 愤怒、不服 |
| 第2擒 | "此乃地利之失，非战之罪！" | "天时不如地利，地利不如人和。" | 辩解、动摇 |
| 第3擒 | "……汝究竟欲如何？" | "欲将军心悦诚服耳。" | 困惑、开始理解 |
| 第4擒 | "纵我七次，我也不降！" | "七乃天地之数，将军且看。" | 倔强、但开始动摇 |
| 第5擒 | "我蛮族男儿，岂能降汉！" | "非降也，同心也。" | 挣扎、部落压力 |
| 第6擒 | "吾之木兽尚且不敌……" | "器不如人，非器之过。" | 技术碾压后的反思 |
| 第7擒 | "……亮公，孟获愿降。" | "将军非降亮，乃降于天下人心。" | 彻底折服 |

## 9. v1.1 设计支柱与边界

### 9.1 三大设计支柱

| 支柱 | 含义 | 必须在游戏中体现为 |
|------|------|------------------|
| **爽快可读的动作体验** | 先让玩家愿意玩，再让玩家愿意听故事 | 操作延迟低、射击反馈明确、敌弹清晰、死亡原因可理解 |
| **擒而不杀的叙事反转** | 每次 Boss 战后都打破“击杀通关”的惯性 | Boss 被击败后进入擒获演出，而非死亡爆炸 |
| **心战递进的角色成长** | 七关不是重复打孟获，而是逐步理解孟获 | 每关孟获必须有新的理由、新的战术、新的动摇 |

### 9.2 非目标（避免跑偏）

| 不做什么 | 原因 |
|----------|------|
| 不做大型开放世界 | 黑客松周期内不可控，且会稀释七擒结构 |
| 不做复杂 RPG 装备系统 | 会抢走“横版射击 + 叙事循环”的焦点 |
| 不做历史百科堆砌 | 历史碎片是奖励，不应阻碍主线节奏 |
| 不把孟获写成纯反派 | 本作核心是归心，不是消灭 |
| 不让 AI 决定主线剧情 | AI 只生成风格化台词，主线必须可控 |

## 10. MVP 范围定义

### 10.1 黑客松优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| P0 | 第1关完整纵切 | 必须完整包含“开场—战斗—Boss—擒获—释放—结算” |

| P0 | 对话与过场系统 | 图文过场 + 战斗内短对话 |
| P0 | 孟获骑象 Boss | 至少 3 种攻击，血量阶段变化 |
| P1 | 故事碎片 | 1 个隐藏碎片即可证明系统 |
| P1 | 关卡评价 | 通关时间、受击次数、碎片收集 |
| P2 | AI 动态台词 | 有 fallback 模板，不影响主流程 |
| P2 | 第2关灰盒 | 可作为加分展示，不要求完整美术 |

### 10.2 第1关原型结构

```
Boot → MainMenu → Level1IntroCutscene
     → Level1Gameplay
     → BossMengHuoElephant
     → CaptureCutscene
     → ReleaseCutscene
     → ResultScene
```

### 10.3 “核心体验已成立”的判断

当试玩者在不阅读说明的情况下，能在通关第 1 关后主动提出以下任一问题，即视为核心体验成立：

- “为什么要放孟获？”
- “下一关他会怎么回来？”
- “诸葛亮到底想干什么？”

## 11. 系统细化

### 11.1 玩家状态机

| 状态 | 描述 | 可切换到 |
|------|------|----------|
| Idle | 站立待机 | Run / Jump / Shoot / Hurt |
| Run | 左右移动 | Idle / Jump / Shoot / Hurt |
| Jump | 空中状态 | Fall / Shoot / Hurt |
| Fall | 下落 | Idle / Run / Hurt |
| Shoot | 射击动作，可与移动叠加 | Idle / Run / Jump |
| Dodge | 短暂无敌翻滚，第2关后解锁 | Idle / Run / Hurt |
| Hurt | 受击硬直 | Idle / Death |
| Death | 死亡 | Retry |
| CutsceneLock | 过场锁定 | Idle / SceneTransition |

### 11.2 武器数值初稿

| 武器 | 解锁 | 射速 | 单发伤害 | 特性 | 风险 |
|------|------|------|----------|------|------|
| 基础弩箭 | 初始 | 中 | 1.0 | 稳定、直线 | 缺少爆发 |
| 连弩 | 第3关 | 快 | 0.6 | 三连发，适合清兵 | 视觉弹幕可能过密 |
| 火箭 | 第5关 | 慢 | 1.5 + DOT | 持续燃烧，克制木兽 | 需要火焰特效区分敌火 |
| 诸葛连弩 | 第7关 | 很快 | 0.8×多方向 | 终章爽感释放 | 需限制弹幕数量避免掉帧 |

### 11.3 敌人设计原则

敌人不只是“血包”，还承担叙事信息：

| 敌人类型 | 玩法功能 | 叙事信息 |
|----------|----------|----------|
| 普通蛮兵 | 基础移动靶 | 孟氏本部勇武直率 |
| 水性蛮兵 | 跳出水面突袭 | 泸水部熟悉地利 |
| 树冠伏兵 | 从遮挡处攻击 | 紫荆山部善伏击 |
| 毒虫/毒雾兵 | 逼迫玩家移动 | 祝融氏掌握毒术 |
| 木兽兵 | 高血量压迫 | 木鹿部驯兽与机关结合 |
| 精英护卫 | 终章高压战斗 | 孟获仍有追随者 |

## 13. AI 动态计谋系统落地方案

### 13.1 输入数据

| 数据 | 示例 | 用途 |
|------|------|------|
| clear_time | 480 秒 | 判断玩家节奏快慢 |
| death_count | 3 | 判断是否需要鼓励或提醒 |
| damage_taken | 7 | 判断是否过于激进 |
| weapon_preference | 基础弩箭 / 连弩 | 生成战术建议 |
| fragment_collected | true/false | 提示探索或赞赏细心 |
| boss_retry_count | 2 | 生成针对 Boss 的谋略提示 |

### 13.2 输出限制

AI 生成内容必须满足：

- 只生成诸葛亮或张伯的短台词，不改动主线剧情。
- 单句不超过 45 个中文字符，战斗内提示不超过 22 个中文字符。
- 不出现现代网络词，不出现明显白话吐槽。
- 不生成历史结局之外的分支。
- 失败时使用本地模板。

## 14. 可用性与体验要求

| 项目 | 要求 |
|------|------|
| 文本速度 | 默认自动推进慢，允许点击加速；首通关键剧情不可完全跳过 |
| 对话位置 | 战斗内对话放右上角或底部，不遮挡玩家和敌弹 |
| 色彩可读性 | 毒雾、火焰、敌弹与背景必须有亮度区分 |
| 失败反馈 | 死亡后提示“被何种机制击败”，避免挫败感不明 |
| 低端设备 | 粒子数量可调，弹幕上限可配置 |
| 无障碍 | 提供屏幕震动强度开关、文字速度、对话回看 |

## 15. 主要风险清单

| 风险 | 等级 | 影响 | 解决方案 |
|------|------|------|----------|
| 七关内容量过大 | 高 | 无法按期完成 | 先做第1关完整纵切，再扩展灰盒 |
| 叙事太长影响爽感 | 中 | 玩家跳过剧情 | 过场短而有钩子，战斗内只放短句 |
| AI 输出不可控 | 中 | 文风不统一 | 固定主线文本，AI 只做非关键短句 |
| Boss 机制重复 | 中 | 后期疲劳 | 每关绑定一种孟获心理变化与一种新机制 |
| 文化表达脸谱化 | 高 | 主题被误解 | 在世界观和角色中强调自治、尊严、技术与信仰 |
| 新手难度过高 | 中 | 首关流失 | 第1关敌弹慢、检查点密、Boss 招式可读 |
