## 核心叙事设计亮点

1. **「擒而不杀」循环**：Boss 战结束 = 叙事开始，这在整个游戏行业是独特的
2. **孟获心态弧线**：傲慢 → 辩解 → 困惑 → 倔强 → 挣扎 → 反思 → 折服（七阶递进）
3. **小兵视角**：汉军无名士卒的视角让「七擒七纵」从宏大叙事变为个人体验
4. **AI 动态台词**：接入混元大模型，根据玩家行为生成诸葛亮计谋分析
5. **色调叙事**：七关色调从翠绿到金棕，反映孟获心态逐步沉淀

## Phase 2 原型范围建议

Phase 2 不建议一开始做 7 关全量内容，而应先做“可证明核心体验”的纵切切片。

### 推荐 MVP：第 1 关完整纵切

| 模块 | 必做 | 可选增强 |
|------|------|----------|
| 基础移动 | 左右移动、跳跃、下落、平台碰撞 | 手柄支持 |
| 基础射击 | 八方向射击、命中反馈、敌人死亡 | 武器掉落 |
| 第1关场景 | 西洱河平原、教学敌人、骑象 Boss | 背景视差 |
| 对话系统 | 开场、战斗中、Boss 前、擒获后 | 打字机效果、头像动画 |
| 释放机制 | Boss 击败后不死亡，进入“擒获—释放”演出 | 玩家短暂不可控镜头 |
| 存档 | 通关第1关、碎片收集 | 评价等级 |
| AI 台词 | 可先用本地模板模拟 | 接入混元 API |

### 第 1 关原型验收标准

- 玩家 30 秒内理解移动、跳跃、射击。
- 玩家首次打败孟获后，必须看到“诸葛亮释放孟获”的反常规叙事。
- 玩家通关后能清楚说出一句话：“我打赢了，但故事还没结束。”
- 对话不应打断战斗节奏；战斗提示必须短于 20 字。
- Boss 不应直接死亡，应从“被击败”切换为“被擒获”。

## 主要风险与规避

| 风险 | 表现 | 规避方式 |
|------|------|----------|
| 玩法与叙事割裂 | 玩家只记得射击，不记得“七擒七纵” | 每次 Boss 后强制出现“擒而不杀”的可视化演出 |
| 诸葛亮过度神化 | 变成“万能军师” | 让他解释逻辑，而不是直接预言结果 |
| 孟获脸谱化 | 只会愤怒和失败 | 每关给他一个新的心理状态和新的作战理由 |
| 历史表达争议 | 把南中部族写成低级反派 | 强化部落文化、自治诉求、技术能力和尊严 |
| 黑客松范围过大 | 7关全做导致半成品 | Phase 2 只做第1关纵切，证明核心循环 |
| AI 功能不稳定 | API 延迟或失败影响体验 | 保留本地模板 fallback，AI 只作为增强层 |

## 最小可展示 Demo 叙事脚本

> 评审试玩时最重要的不是展示所有关卡，而是让他们在 8 分钟内理解本作的独特性。

```
开场：诸葛亮说明“此战所求，非杀戮，乃归心”
  ↓
玩家横版射击推进，学习基础操作
  ↓
孟获骑象登场，表现为传统 Boss 战
  ↓
玩家击败孟获，以为通关
  ↓
镜头切到诸葛亮：松绑，释放孟获
  ↓
孟获震惊离去，玩家产生疑问
  ↓
结算界面提示：第一擒完成，第二擒解锁
```

这个 Demo 只要完成，就已经证明了项目的核心差异化：**Boss 战不是故事终点，而是下一次叙事循环的起点。**
