## 2. 研发工作流（7个阶段）

### Phase 1 — 产品定义 & 叙事设计（Week 1）

**核心任务：**
- 确定游戏核心体验主张：叙事 × 动作的融合方式
- 编写世界观文档：七擒背景、蛮夷文化设定、历史还原度取舍
- 设计7关卡叙事弧线（每关抓获孟获后的剧情走向）
- 撰写角色设定表：诸葛亮、孟获、蛮军将领、汉军士卒
- 确定核心机制：横版射击 + 叙事对话节点的结合方式

**交付物：**
- 游戏设计文档 GDD（Game Design Document）
- 世界观圣经（World Bible）
- 关卡叙事大纲（7关各一页）
- 角色设定表（主角 + 主要NPC）

**关键决策（本阶段必须确认）：**
- 技术栈选型（见第4章）
- 画风方向：像素风 or 水墨国风
- 叙事介入方式：对话框 / 过场动画 / 两者结合
- 玩家视角：扮演诸葛亮本人？汉军将领？无名士卒？

---

### Phase 2 — 原型验证：核心玩法 Demo（Week 2）

**核心任务：**
- 搭建项目工程框架（游戏引擎 + 前端工程初始化）
- 实现玩家基础操控：移动、跳跃、射击（子弹型投射物）
- 制作第1关灰盒原型（用方块代替美术资源验证关卡结构）
- 实现最小叙事系统：对话框触发、文字展示、跳过功能
- 关卡完成判定逻辑 + 孟获"被擒"触发事件

**交付物：**
- 可玩灰盒 Demo（第1关可通关）
- 基础操控系统（移动/跳跃/射击）
- 对话系统 v1（触发 + 展示 + 跳过）

**验收标准：**
- 手感流畅，移动和射击无明显卡顿
- 对话节点能在正确时机触发并可跳过
- 能完整通关第1关并展示叙事结局画面

### Phase 3 — 美术 & 音效资产制作（Week 2–3，与 Phase 2 并行）

**核心任务：**
- 角色像素精灵动画：诸葛亮（主角）、孟获（Boss）、蛮兵、汉兵
- 7套关卡环境图块集（Tileset）：丛林、山地、营寨、洞穴、河流等
- UI 元素制作：血条、弹药计数、对话框边框、过关界面（三国国风）
- 背景音乐创作：战鼓 + 古琴氛围，7关各一首（风格递进）
- 音效库：射击音、击中音、对话提示音、通关音、环境音

**工具推荐：**
- 像素美术：Aseprite（付费，专业）/ Pixilart（免费，在线）
- UI 设计：Ardot（已集成于本项目工作流）
- 背景音乐：Suno AI（AI生成三国风音乐）/ LMMS（免费DAW手作）
- 音效：freesound.org / BFXR（像素音效生成器）

**交付物：**
- Sprite 精灵表（含各动画帧：行走、跳跃、射击、死亡）
- Tileset 环境图块 × 7套
- UI 素材包（PNG + 源文件）
- BGM × 7首（WAV/OGG格式）
- 音效库（分类整理）

### Phase 4 — 7关卡全量开发（Week 3–5）

**每关标准结构：**

```
开场叙事（1–2分钟）
  └─ 战前诸葛亮对话 / 军情情报
关卡主体（5–10分钟）
  └─ 地图布局 + 敌兵配置 + 地形障碍
Boss战：孟获
  └─ 专属攻击模式（每关递增难度）
结局叙事（1分钟）
  └─ 擒获动画 + 孟获反应台词 + 孟获逃脱
过关奖励
  └─ 解锁下一章叙事碎片
```

**7关开发安排：**

| 关卡 | 主题 | 场景 | 核心机制 |
|------|------|------|---------|
| 第1擒 | 初战锋芒 | 西洱河平原 | 入门教程，基础射击 |
| 第2擒 | 泸水险滩 | 泸水河岸 | 跳跃挑战，水流障碍 |
| 第3擒 | 深入蛮营 | 紫荆山蛮营 | 潜入 + 射击混合 |
| 第4擒 | 毒雾弥漫 | 秃龙洞洞窟 | 环境障碍，视野限制 |
| 第5擒 | 联军围攻 | 蛮族联盟营地 | 多波次敌兵，精英兵种 |
| 第6擒 | 木兽火攻 | 蛮夷要塞 | 特殊机关，火焰障碍 |
| 第7擒 | 银坑洞决战 | 孟获王庭 | 最终Boss，全机制综合 |

**里程碑检查点：**
- 每关完成后进行内部玩测
- 记录 Bug 清单和操控手感反馈
- 难度曲线评估（是否有明显跳跃）

### Phase 5 — 叙事系统深化 & 剧情打磨（Week 4–5，与 Phase 4 并行）

**交付物：**

## 3. 工作流核心逻辑

### 3.1 最重要的原则：先定义，再动手

**黑客松叙事赛道的评审标准是「叙事与玩法是否真正融合」**，而不是关卡数量或技术复杂度。第一周投入在 GDD 上，是后续所有工作的地基。

GDD 必须回答的三个核心问题：

> **Q1：诸葛亮为什么选择「七擒」而非直接歼灭？**
> 答案决定了游戏的核心价值观，也驱动了"擒获后释放"的关卡循环机制。

> **Q2：孟获每次被擒后的心态如何变化？**
> 答案决定了叙事弧线和难度递增的合理性——Boss 变强是因为孟获越来越认真对待诸葛亮。

> **Q3：玩家扮演谁？**
> 视角直接决定代入感。扮演诸葛亮 = 策略感强；扮演无名汉军士卒 = 沉浸感强，更易共情。

### 3.2 关卡设计的叙事驱动逻辑

每关不应只是"场景不同的同一款游戏"，而应体现叙事进展带来的机制变化：

```
第1擒：孟获轻视汉军 → 关卡简单，孟获几乎不认真
第2擒：孟获开始布局 → 引入环境障碍
第3擒：孟获调兵遣将 → 敌兵种类增加
...
第7擒：孟获倾尽全力 → 综合所有机制的最终挑战
```

### 3.3 黑客松节奏管理

- **Week 1 产出 GDD 是硬性里程碑**，没有 GDD 就不应开始写代码
- **Week 2 必须有可玩 Demo**，哪怕只有第1关且全是方块
- **每关完成后立即玩测**，别等到全部7关做完再测，那时调整成本极高
- **最后1周留给打磨**，技术债在这周还，不要在最后1周加新功能

## 4. 技术栈建议

### 4.1 推荐方案对比

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Phaser 3 + TypeScript** | 纯前端Web运行，部署最简单，社区资源丰富，黑客松最快速 | 大型项目维护成本高 | ⭐⭐⭐⭐⭐ **首选** |
| **Cocos Creator** | 腾讯生态，CloudBase 集成顺畅，移动端成熟 | 工程较重，初始配置复杂 | ⭐⭐⭐⭐ |
| **Godot 4（导出Web）** | 功能最全，引擎内置叙事工具 | 导出Web构建较大，加载慢 | ⭐⭐⭐ |

**结论：选 Phaser 3 + TypeScript**，理由是：
1. 零安装，浏览器直接运行，评审官体验零门槛
2. CloudBase 静态托管完全支持
3. 社区有大量类魂斗罗的开源示例可参考

## 6. 叙事赛道差异化建议

### 6.1 核心差异化：「七擒七纵」本身就是最强叙事钩子

大多数动作游戏的 Boss 打倒即结束。《七擒孟获》的独特性在于：**打倒 Boss 只是故事的开始**。

这个「擒而不杀、纵而再战」的循环，天然构成了游戏最强的叙事张力：

- **玩家角度**：为什么我打赢了却不结束？→ 引发对诸葛亮战略意图的好奇
- **孟获角度**：为什么打败了还被放走？→ 触发心理变化的叙事空间
- **文化角度**：「心战为上，攻城为下」→ 东方智慧的现代游戏化表达

**建议在关卡设计中强化这个钩子：**
- 擒获孟获后，屏幕不出现"通关"，而出现孟获的愤慨台词
- 诸葛亮亲自下令释放，并附上一句意味深长的话
- 下一关开场时，孟获带着新的仇恨卷土重来，关卡难度随之提升

### 6.2 技术创新加分点：AI 生成诸葛亮"计谋"台词

腾讯云黑客松天然鼓励云服务整合。建议接入**腾讯云混元大模型（HunyuanLLM）**：

**玩法设计：**
- 在每关开场，根据玩家上一关的游玩风格（激进/稳健/快速/缓慢）
- AI 动态生成一段诸葛亮的"计谋分析"台词
- 台词内容提及玩家的具体行为，增强代入感

**示例：**
> 玩家上关激进速通 → 诸葛亮："将军锐气可嘉，然穷寇莫追。此关孟获布有伏兵，宜稳中取胜。"
> 玩家上关死亡多次 → 诸葛亮："蛮夷地势熟稔，我军初来难免受挫。此战当以地形为先，切勿冒进。"

**技术实现：**
```typescript
// 调用腾讯云混元API
const generateZhugeLiangAdvice = async (playerStats: PlayerStats) => {
  const prompt = buildPrompt(playerStats); // 根据游玩数据构建 Prompt
  const response = await hunyuanClient.chat(prompt);
  return response.content; // 返回台词文本
};
```

### 6.3 文化深度展现策略

叙事赛道的评审通常关注「文化表达是否有深度，而非贴标签」：

| 浅层文化（避免）| 深层文化（追求）|
|--------------|--------------|
| 关卡名字叫三国地名 | 关卡机制反映历史战术特点 |
| 角色穿古装 | 角色台词体现身份和价值观差异 |
| BGM 用古筝 | 音乐随叙事情绪变化 |
| "南蛮"作为单纯反派 | 蛮夷文化有其内在逻辑和尊严 |

**具体建议：**
- 给孟获设计一套「蛮族义气」的行事逻辑，而非单纯的"强大反派"
- 蛮兵不只是炮灰，每关出现特定蛮族将领，有简短背景介绍
- 第7关擒获后，孟获的折服台词应是全游戏最有分量的一段文字——值得在这里花时间

### 6.4 故事碎片收集系统（强烈建议加入）

设计7个可收集的「历史碎片」，通关每关解锁一块，集齐7块后展示完整的《出师表》段落或真实史书中关于七擒孟获的记载。

优点：
- 增加重玩价值
- 将游戏与真实历史文化连接
- 评审官看到"尊重史实"的态度
