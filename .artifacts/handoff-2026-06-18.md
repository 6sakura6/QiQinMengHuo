## 4. 场景流转（已验证闭环）

```
BootScene ──▶ MainMenuScene ──▶ Level1Scene ──▶ ResultScene ──▶ MainMenuScene
```

- Boot → MainMenu：直接 `scene.start('MainMenuScene')`
- MainMenu → Level1：`fadeOut(250ms)` → `delayedCall(260ms)` → `scene.start('Level1Scene')`
- Level1 → Result：CAPTURE_COMPLETE 事件 → `delayedCall(800ms)` → `scene.start('ResultScene', {...})`
- Result → MainMenu：直接 `scene.start('MainMenuScene')`
- 四个场景全部注册在 `gameConfig.ts` scene 数组

## 5. 关键架构决策 & 陷阱

### 5.1 三层安全网（Level1Scene）

由于 Phaser TimerManager 帧序竞争问题，Level1Scene 内置三层防御：

| 层 | 触发条件 | 行为 |
|----|----------|------|
| 1 | CAPTURE_COMPLETE 后 2s 未进 ResultScene | 强制 `scene.start('ResultScene')` |
| 2 | 擒获启动后无对话 > 5s | 强制触发 CAPTURE_COMPLETE + 跳转 |
| 3 | `_dialogActive` 卡住 > 12s | 强制 `physics.resume()` + `_dialogActive = false` |

> **不要移除这些安全网**，除非 TimerManager 帧序竞争问题从架构层面解决。

### 5.2 Boss 对话延迟自管理（替代 Phaser TimerEvent）

```typescript
// 不要用 this.time.delayedCall() 做 Boss 登场对话延迟！
// TIMERMANAGER.UPDATE() 先于用户 UPDATE() 执行，会导致帧序竞争。
// 正确做法：自管理 delta 计数
private _bossIntroDelayMs = 0;

update(time, delta) {
  if (bossActive && !captureStarted && !boss.isDefeated && _bossIntroDelayMs > 0) {
    _bossIntroDelayMs -= delta;
    if (_bossIntroDelayMs <= 0) { /* 触发 BOSS_INTRO */ }
  }
}
```

### 5.3 hasAnyProgress() 修复

`blankSave()` 初始化 `unlockedLevels: ['level_01']`，必须排除默认关卡才能判断真实进度：

```typescript
hasAnyProgress(): boolean {
  return this._data.unlockedLevels.length > 1  // 排除默认 level_01
      || this._data.collectedFragments.length > 0
      || Object.keys(this._data.levelRecords).length > 0;
}
```

### 5.4 MainMenuScene 输入锁

重置确认期间 `inputLocked = true` 禁用键盘导航/选择，3s 超时或确认后释放。

## 6. 下一任务：Batch 11

**AudioManager + 基础音效占位**

参考架构文档第 3.1 节 AudioManager 接口定义 `src/systems/AudioManager.ts`：

```typescript
interface IAudioManager {
  playSfx(key: string, config?: { volume?: number; loop?: boolean }): void;
  playBgm(key: string): void;
  stopBgm(): void;
  setMasterVolume(volume: number): void;
  toggleMute(): void;
}
```

**核心要求**：
- 所有方法必须有空实现（资源未加载时静默忽略，不抛异常）
- `gameConfig.ts` scene 数组不需修改
- Level1Scene 中 Boss 攻击、玩家受击、对话等位置调用 `AudioManager.playSfx()` 占位
- 完成标准：`npm run build` 零错误，游戏中不会因为没有音频文件而崩溃
